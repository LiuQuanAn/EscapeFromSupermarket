# 货架物品识别功能规范

日期：2026-05-24

状态：待实现

关联设计：`docs/gameplay-design.md` 第 8 章、`docs/design-log.md` 第 33 轮。

## 1. 问题陈述

当前货架打开后，商品信息会直接展示。这样玩家能立即判断价值、重量、格子和任务标记，货架交互更像普通菜单选择，缺少搜打撤中“冒风险识别未知物品”的压力。

本功能要让玩家打开货架后先看到未知物品，并通过逐个读条识别信息。玩家需要在保安巡逻、倒计时、撤离压力和货架 UI 实时运行的情况下决定是否继续等待识别，还是关闭货架离开。

## 2. 目标

- 让货架交互从“立即选择商品”变成“冒风险识别商品后再决策”。
- 让稀有物品带来更长识别时间，从而增加时间风险。
- 让玩家每轮需要重新识别货架，强化本轮内的探索和记忆。
- 保持商品拿取瞬间完成，不重新引入拿取读条。
- 增加必要 UI 反馈，让玩家清楚当前正在识别什么、还剩多久、哪些物品已识别。

## 3. 非目标

- 不做完整图鉴系统。
- 不做跨轮永久识别记忆。
- 不做远程扫描。
- 不做盲拿未知物品。
- 不改变商品拿取瞬间完成的规则。
- 不新增货架外的复杂扫描工具。
- 不新增多层教程 UI。

## 4. 用户故事

1. 作为玩家，我想打开货架后先看到未知物品，这样搜刮时有未知风险。
2. 作为玩家，我想看到当前物品的识别进度，这样我知道还要等多久。
3. 作为玩家，我想看到读秒和进度条，这样我能在压力下快速判断是否继续等。
4. 作为玩家，我想一次只识别一个物品，这样货架识别有明确节奏。
5. 作为玩家，我想已识别物品在本轮内保持已识别状态，这样我能离开后回来继续利用信息。
6. 作为玩家，我想未识别完成的物品下次从头识别，这样中途离开有明确代价。
7. 作为玩家，我想新一轮重新识别所有物品，这样每轮都有探索感。
8. 作为玩家，我想稀有物品识别更久，这样高价值信息有更高时间成本。
9. 作为玩家，我想识别完成后再拿取商品，这样不会误拿未知物品。
10. 作为玩家，我想货架 UI 在识别时仍不暂停游戏，这样保安和倒计时压力不会断掉。

## 5. 核心逻辑

### 5.1 货架打开状态

玩家打开货架后，货架列表中每个商品槽位有两种可见状态：

- 未识别：只显示未知占位，不显示商品名、价值、重量、格子、风险或任务标记。
- 已识别：显示完整商品信息，并允许玩家拿取。

货架 UI 打开时，游戏不暂停，不慢动作。

### 5.2 逐个识别

识别按货架列表顺序逐个进行。

规则：

- 同一时间只识别 1 个商品。
- 当前商品识别完成后，才开始识别下一个未识别商品。
- 已识别商品跳过，不重复读条。
- 若货架里所有商品均已识别，则不再显示识别读条。
- 玩家不能手动并行识别多个商品。

### 5.3 识别时间

识别时间由商品稀有度决定。稀有度越高，时间越长。

建议初始调参：

| 稀有度 | 识别时间 |
| --- | ---: |
| 普通 | 1 秒 |
| 稀有 | 2 秒 |
| 高稀有 | 5 秒 |
| 任务 | 3 秒 |

这些数值是首轮调参起点，不是最终平衡值。

### 5.4 本轮内进度保存

识别完成状态只在本轮内保存。

规则：

- 本轮已经识别完成的商品，再次打开同一货架时仍为已识别。
- 本轮未识别完成的商品不保存部分进度。
- 如果玩家关闭货架、离开交互范围、被打断或本轮失败，当前未完成识别项下次从头开始。
- 新一轮开始时，所有货架商品都恢复未识别状态。
- 识别状态绑定本轮货架内的商品实例，不跨轮保存。

### 5.5 拿取关系

识别和拿取是两个独立动作。

规则：

- 识别读条只负责显示商品信息。
- 商品拿取仍然是瞬间动作。
- 未识别商品不可拿取。
- 已识别商品可以按现有规则一次拿取。
- 容量不足、重量超限或其他条件不满足时，按现有拿取失败提示处理。

## 6. UI 规范

### 6.1 未识别项

未识别商品显示为未知占位。

建议显示：

- `未知物品`
- 或 `???`
- 或带外星扫描器风格的占位图标

不得显示：

- 商品名
- 价值
- 重量
- 格子
- 稀有度
- 任务标记

### 6.2 当前识别项

当前正在识别的商品槽位需要显示：

- 识别中状态。
- 剩余时间读秒。
- slider 进度条。

示例：

`识别中 1.2s`

进度条从 0 到 100%，与剩余时间同步。

### 6.3 已识别项

已识别商品显示完整信息：

- 商品名
- 价值
- 格子
- 重量
- 任务标记，如有
- 风险或特殊标记，如有

已识别项可执行拿取。

### 6.4 中断反馈

如果玩家在当前物品识别完成前关闭货架或离开范围：

- 不弹复杂提示。
- 当前物品下次重新识别。
- 可用轻量状态提示，例如 `识别中断`。

## 7. 预期体验

玩家打开货架后，不能立刻知道里面有哪些商品。玩家会看到未知项依次被识别。

预期决策：

- 只等前几个普通物品，还是冒风险继续等后面的稀有物品。
- 保安靠近时，是继续识别当前物品，还是立刻关 UI 离开。
- 本轮已经识别过的货架，是否值得回来拿货。
- 看到高价值物品后，是否为了容量和重量调整购物车。

该功能应增加“等信息”的风险，但不应让货架交互变得拖沓。

## 8. 边界条件

- 玩家打开空货架时，不显示识别读条。
- 商品已经被拿走后，不再出现在货架中。
- 已识别但未拿走的商品，本轮内再次打开仍显示完整信息。
- 当前识别项如果被拿走、货架刷新或本轮结束，识别流程停止。
- 倒计时结束时，货架 UI 关闭并进入失败结算。
- 保安抓到玩家时，货架 UI 关闭并进入失败结算。
- 识别读条不影响玩家移动系统之外的全局时间。

## 9. 验收标准

功能完成后，应满足：

- 打开货架后，未识别商品不显示真实信息。
- 商品按顺序逐个识别。
- 当前识别项显示倒计时读秒和 slider 进度。
- 不同稀有度使用不同识别时间。
- 已识别商品本轮内再次打开保持已识别。
- 未识别完成的商品关闭后再次打开从头识别。
- 新一轮开始后所有商品重新变为未识别。
- 未识别商品不可拿取。
- 已识别商品仍按瞬间拿取规则进入购物车。
- 游戏在货架 UI 打开时不暂停，保安、倒计时和撤离状态继续运行。

## 10. 测试决策

测试应关注外部行为，不测试内部实现细节。

需要覆盖：

- 识别顺序。
- 识别完成状态保存。
- 未完成进度不保存。
- 新一轮重置识别状态。
- 不同稀有度识别时间差异。
- 已识别后拿取。
- 未识别时禁止拿取。
- 货架 UI 打开时倒计时和保安继续运行。

## 11. 本轮不处理的问题

- 是否允许盲拿未知商品。
- 是否允许玩家手动选择优先识别哪个未知项。
- 是否把识别信息保存到长期图鉴。
- 是否通过角色成长缩短识别时间。
- 是否通过扫描器升级远程识别货架。
- 是否让任务商品有独立识别音效或特效。
