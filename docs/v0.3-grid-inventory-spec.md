# V0.3 格子库存系统规范

日期：2026-05-30

状态：待实现

来源：基于 `docs/gameplay-design.md`、`docs/v0.2-vertical-slice-spec.md`、`docs/shelf-item-identification-spec.md` 和已通过的 V0.2 playtest 制定。

## 1. 目标

V0.3 目标是把购物车容量从抽象格子数升级为二维格子库存，让“商品形状、摆放位置、旋转、腾空间”成为搜刮决策的一部分。

本轮不是扩充大量商品内容，而是升级物品系统和库存 UI，使玩家在拿货时需要同时考虑：

- 商品价值。
- 商品重量。
- 商品形状。
- 购物车剩余空间。
- 是否值得丢弃低价值物品腾位置。
- 未识别物品被读出后带来的尺寸惊喜或失望。

## 2. 核心体验

玩家打开货架后，先逐个识别未知物品。物品识别完成后，显示真实商品、价值、重量和占用形状。玩家尝试拿取时，需要把该物品放进购物车二维格子中。放得下则瞬间拿取，放不下则不能拿。

预期效果：

- 玩家不再只看总格子数，而要判断空间形状。
- 高价值物品可能因为形状太大或太别扭而被放弃。
- 玩家会主动丢弃低价值物品，给高价值或任务物品腾空间。
- 重量系统继续保留，避免库存只剩“拼图”决策。
- UI 比当前抽象列表更直观，但仍服务于快速搜刮。

## 3. 非目标

V0.3 不做：

- 角色背包。
- 胸挂、口袋、装备栏。
- 多容器系统。
- 容器套容器。
- 保险箱 UI。
- 自动整理。
- 商品堆叠。
- L 形、多边形或不规则形状。
- 耐久、品质、损坏。
- 拆解、研究、出售分流界面。
- 完整 Tarkov 式装备系统。

## 4. 用户故事

1. 作为玩家，我想看到购物车的二维格子，这样我能直观理解剩余空间。
2. 作为玩家，我想看到物品的形状，这样我能判断它是否放得下。
3. 作为玩家，我想旋转物品，这样我能优化购物车空间。
4. 作为玩家，我想拖动物品到购物车里，这样拿货有明确摆放动作。
5. 作为玩家，我想非法位置显示红色，这样我知道为什么放不下。
6. 作为玩家，我想合法位置显示绿色，这样我能快速确认摆放。
7. 作为玩家，我想丢弃已装商品，这样我能为更有价值的商品腾空间。
8. 作为玩家，我想未识别商品不暴露真实形状，这样识别信息有价值。
9. 作为玩家，我想任务商品也受形状和重量限制，这样任务目标有真实成本。
10. 作为玩家，我想重量仍影响移动，这样格子系统不会取代负重压力。

## 5. 核心规则

### 5.1 购物车网格

购物车使用二维格子。

建议初始尺寸：

- 基础购物车：8 列 x 6 行。

每个格子只能被一个商品占用。商品必须完整放入购物车网格内，不能越界，不能和其他商品重叠。

购物车仍保留承重上限。网格决定“放不放得下”，重量决定“推不推得动”。

### 5.2 商品形状

V0.3 只做矩形商品。

商品需要新增尺寸字段：

- 宽度：占用列数。
- 高度：占用行数。

示例尺寸：

| 类型 | 示例 | 尺寸 |
| --- | --- | --- |
| 小件 | 薯片、牙膏、电池 | 1x1 |
| 细长件 | 法棍、雨伞、长包装 | 1x2 |
| 中件 | 洗衣液、罐装礼盒 | 2x2 |
| 中大件 | 路由器、空气炸锅 | 2x3 |
| 扁大件 | 微波炉 | 3x2 |
| 大件 | 电视 | 4x3 |

实际商品尺寸可以后续调参，但必须能覆盖小件、中件、大件差异。

### 5.3 旋转

商品可以旋转 90 度。

规则：

- 旋转后宽高互换。
- 旋转只影响摆放形状，不影响价值、重量、稀有度或任务标记。
- 正方形商品旋转后视觉上可不变化。
- 当前默认所有矩形商品都允许旋转。

后续可以扩展为部分商品禁止旋转，例如液体、蛋糕、易碎物，但 V0.3 不做。

### 5.4 放置

玩家把已识别商品放入购物车网格。

合法放置条件：

- 商品所有占用格都在购物车边界内。
- 商品所有占用格都为空。
- 放入后总重量不超过购物车承重上限。

非法放置原因：

- 越界。
- 与已有商品重叠。
- 超重。

非法时商品不能拿取。

### 5.5 拿取

商品拿取仍然是瞬间动作。

新规则：

- 已识别商品才能尝试拿取。
- 玩家选择放置位置后，合法则商品立刻进入购物车。
- 不做拿取读条。
- 不做中断。
- 不做二次确认。
- 不做自动寻找位置。

### 5.6 丢弃

玩家可以从购物车中丢弃商品。

规则：

- 丢弃后该商品占用格立即释放。
- 丢弃后该商品重量立即从购物车重量中移除。
- V0.3 中丢弃商品直接消失。
- 不生成地面掉落物。
- 不触发 NPC 反应。

### 5.7 重量

重量系统保留。

规则：

- 商品仍有重量。
- 购物车仍有承重上限。
- 超重不能放入新商品。
- 当前重量继续影响移动速度或负重档位。
- UI 需要同时显示空间占用和重量状态。

## 6. 货架与识别关系

### 6.1 未识别商品

未识别商品不显示真实信息。

不得显示：

- 名称。
- 价值。
- 重量。
- 尺寸。
- 形状。
- 任务标记。

可显示：

- `未知物品`。
- 识别进度。
- 读取倒计时。

### 6.2 已识别商品

识别完成后显示：

- 名称。
- 价值。
- 重量。
- 尺寸。
- 形状预览。
- 稀有度。
- 任务标记，如有。

已识别商品才允许进入放置流程。

### 6.3 识别状态

沿用现有货架识别规则：

- 同一时间只识别 1 个商品。
- 逐个读取。
- 稀有度越高，读取时间越长。
- 已识别状态本轮保存。
- 未读完进度不保存。
- 新一轮全部重置。

## 7. UI 规范

### 7.1 总体布局

货架 UI 打开时，显示两块主要区域：

- 左侧：货架商品列表。
- 右侧：购物车网格。

游戏不暂停，不慢动作。NPC、倒计时、警戒继续运行。

### 7.2 货架列表

货架列表显示当前货架的商品槽位。

未识别项：

- 显示未知占位。
- 当前读取项显示倒计时和 slider。

已识别项：

- 显示商品名。
- 显示价值。
- 显示重量。
- 显示尺寸，例如 `2x3`。
- 显示任务标记，如有。
- 鼠标悬停或选中时显示形状预览。

### 7.3 购物车网格

购物车网格显示：

- 空格。
- 已放入商品。
- 商品占用范围。
- 商品名称或简短缩写。
- 当前拖拽物品预览。

视觉反馈：

- 合法放置：绿色高亮。
- 非法放置：红色高亮。
- 超重：重量区域红色提示。
- 重叠：冲突格红色提示。
- 越界：超出部分或边界红色提示。

### 7.4 操作

基本操作：

- 点击或拖拽已识别商品，进入放置状态。
- 鼠标移动选择放置位置。
- 左键确认放置。
- 右键或 Esc 取消放置。
- `R` 旋转当前拖拽物品。
- 点击购物车内商品可选择。
- 选择购物车内商品后可丢弃。

具体按键可按项目现有输入习惯调整，但 UI 必须明确提示旋转和取消。

### 7.5 常驻 UI

常驻 UI 仍保持精简。

需要显示：

- 当前重量 / 承重上限。
- 当前估值。
- 当前负重档位或移动惩罚。

原来的抽象格子占用可以替换为：

- 已用格子 / 总格子。

但主要空间管理应在购物车网格 UI 中完成。

## 8. 数据规则

每个商品至少需要这些玩法字段：

- 商品 ID。
- 名称。
- 价值。
- 重量。
- 宽度。
- 高度。
- 稀有度。
- 货架类别。
- 是否任务商品。

购物车内每个商品实例至少需要这些状态：

- 商品 ID。
- 当前左上角格子坐标。
- 当前旋转状态。
- 当前占用宽度。
- 当前占用高度。

货架内每个商品实例继续保留识别状态。

## 9. 边界条件

- 购物车满但仍有空洞时，只要商品形状能放入空洞，允许拿取。
- 已识别商品如果被拿走，从货架移除。
- 拿取失败不移除货架商品。
- 关闭货架 UI 时，已放入购物车的商品保留。
- 关闭货架 UI 时，正在拖拽但未放置的商品取消，不进入购物车。
- 关闭货架 UI 时，未读完识别进度按识别规则重置。
- 回合失败后，购物车内商品按现有失败规则丢失。
- 回合成功后，购物车内商品按现有结算规则结算。
- 新一轮开始后，购物车清空。
- 商品旋转不能让已放置商品重叠或越界。

## 10. 与现有系统关系

### 10.1 替换抽象格子

V0.3 后，商品不再只用一个抽象 `slots` 数值决定容量。

旧规则：

- 当前格子 + 商品格子 <= 容量。

新规则：

- 商品形状能放入二维网格。
- 放入后总重量不超过承重上限。

### 10.2 保留重量

重量仍是独立压力来源。

即使商品形状能放下，如果重量超限，也不能拿。

### 10.3 保留识别系统

未识别商品不显示形状。识别完成后才暴露尺寸和放置需求。

### 10.4 保留瞬间拿取

放置合法后，商品立即进入购物车。格子放置流程不是拿取读条。

## 11. 调参起点

建议基础购物车：

- 8 列 x 6 行。
- 承重沿用当前版本，后续根据 playtest 调整。

建议商品尺寸：

| 商品 | 尺寸 |
| --- | --- |
| 薯片 | 1x1 |
| 罐头 | 1x1 |
| 牙膏 | 1x1 |
| 洗衣液 | 2x2 |
| 路由器 | 2x2 |
| 微波炉 | 3x2 |
| 电视 | 4x3 |

建议先用现有商品池验证形状决策，不急着扩充大量新商品。

## 12. 验收标准

V0.3 完成后，应满足：

- 购物车显示二维格子。
- 商品有宽高形状。
- 已识别商品显示形状和尺寸。
- 未识别商品不显示真实形状。
- 玩家可把商品放入购物车网格。
- 合法位置绿色提示。
- 非法位置红色提示。
- 越界、重叠、超重时不能拿取。
- 玩家可旋转物品。
- 玩家可丢弃购物车内商品。
- 丢弃后空间和重量立即释放。
- 放置合法后，商品瞬间进入购物车。
- 游戏在库存 UI 打开时不暂停。
- 现有货架识别规则继续有效。
- 现有结算、撤离、失败规则继续有效。

## 13. Playtest 关注点

测试 5 到 10 轮，观察：

- 玩家是否会因为形状放弃高价值商品。
- 玩家是否会为了任务商品重新整理或丢弃商品。
- 玩家是否理解旋转。
- 玩家是否理解红绿放置反馈。
- 未识别商品不显示形状是否带来合理悬念。
- UI 是否过慢，是否破坏实时压力。
- 重量是否仍有存在感。
- 8x6 网格是否太大或太小。
- 电视、微波炉等大件是否有足够决策压力。

## 14. 本轮不处理的问题

- 自动整理。
- 多容器。
- 背包、胸挂、口袋。
- 保险箱。
- 商品堆叠。
- L 形物品。
- 格子内物品旋转动画。
- 地面掉落物。
- 长按快速丢弃。
- 控制器手柄操作。
- 存档中的库存持久化。
- 局外仓库库存。
