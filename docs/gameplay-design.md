# 超市搜打撤游戏玩法设计文档

日期：2026-05-23

来源：本文档由 `docs/design-log.md` 的逐轮决策整理而来。`design-log.md` 保留为原始决策记录，本文档作为系统化玩法设计入口。

## 1. 项目概述

本项目是一款以超市为主题的单人 PvE 搜打撤游戏。玩家扮演一名误解地球购物规则的外星人，推着购物车进入超市，在关店倒计时和 NPC 干扰下尽可能拿走高价值或高用途商品，并成功撤离。每轮带出的商品可用于赚钱、拆解、研究、交付任务或修复飞碟，从而推进长期成长。

### 1.1 核心卖点

- 超市抢购题材与搜打撤结构结合。
- 购物车既是容量系统，也是核心构筑载体。
- 商品价值不只看价格，还看重量、格子、用途、风险和撤离成本。
- NPC 不是单纯敌人，而是超市规则、路线压力和场内变量。
- 外星人误解地球文化带来荒诞喜剧包装，但底层玩法压力认真。

### 1.2 设计原则

- 玩家优化的是风险调整后的收益，不是单纯拿最贵商品。
- 拿取动作保持直接，决策压力放在选择、装载、移动、警戒和撤离。
- 地图可学习，但每轮通过库存、NPC、撤离条件和事件制造变化。
- 失败不归零，保留信息、部分任务进度、部分经验和保险箱物品。
- 原型先验证单轮是否好玩，局外系统只做最薄闭环。

## 2. 世界观与语气

主角是一名外星人。由于飞碟故障，主角临时停靠地球，并收看到地球上的“限时在超市里拿东西”的综艺节目。主角误以为地球物资就是通过这种方式获得的，于是开始反复进入超市搜刮物资，试图修好飞碟并回到母星。

游戏主语气是荒诞喜剧，但系统压力认真。荒诞感来自外星人对超市、商品、综艺节目、小票、会员卡、试吃和员工区等地球规则的误解。失败表现偏综艺淘汰或卡通化处理，不走血腥死亡方向。

## 3. 核心循环

### 3.1 单轮循环

1. 选择基础购物车配置和少量局外升级。
2. 进入固定大布局、随机变量的超市。
3. 观察货架类型、NPC 巡逻、撤离口和倒计时。
4. 打开货架 UI，通过外星扫描器查看具体商品。
5. 根据价值、重量、格子、风险、任务和撤离路线拿商品。
6. 管理购物车容量、重量、可疑程度和移动手感。
7. 在倒计时结束前满足撤离条件并完成撤离。
8. 结算商品，选择出售、拆解、交付、研究或存入保险箱。
9. 使用收益强化购物车、角色或飞碟模块。

### 3.2 长期循环

长期目标是修复飞碟。飞碟模块既是剧情进度，也是系统解锁来源。

可拆分模块：

- 能源核心：需要电池、发电机、压缩机等。
- 导航系统：需要电视、路由器、收音机、电子零件等。
- 船体修复：需要胶带、铝箔、工具箱、建材等。
- 生命维持：需要空气净化器、鱼缸泵、冷藏设备等。
- 伪装系统：需要节日装饰、广告灯箱、超市制服、纸箱等。

模块修复后可解锁扫描强化、撤离口预测、夜间超市、更大保险箱或特殊购物车模块。

## 4. 单轮结构

### 4.1 地图结构

每轮采用固定大布局加随机变量。

固定区域示例：

- 入口
- 零食区
- 日用品区
- 冷冻区
- 生鲜区
- 家电区
- 仓库
- 员工通道
- 收银区
- 装卸口

随机变量：

- 货架库存
- 稀有商品位置
- 任务商品位置
- 保安巡逻线
- 店员站位
- 清洁机器人路线
- 撤离条件
- 特殊事件

目标是让玩家能学习地图，但每轮仍要侦察和临场判断。

### 4.2 倒计时

关店倒计时先保持简单，作为硬时间限制。原型阶段不做倒计时分阶段改变规则。倒计时主要迫使玩家在继续搜刮和及时撤离之间取舍。

## 5. 视角与控制

### 5.1 视角

游戏采用第三人称俯视角，参考《逃离鸭科夫》一类俯视搜打撤游戏的读图方式。

视角必须清楚表达：

- 购物车位置和朝向
- 通道宽度
- 货架关系
- NPC 视线
- 巡逻路线
- 人群阻挡
- 撤离方向

### 5.2 人车控制模型

默认状态下，玩家直接控制“推车状态的角色 + 购物车整体”，不是角色自由走、购物车在身后拖拽。

控制要点：

- 购物车重量影响整体速度、转向、刹车和碰撞声音。
- 小商品可以在靠近货架时快速选择并入车。
- 中型、大型和特殊商品可以要求停车、低速或满足槽位条件，但拿取本身瞬间完成。
- 特殊交互可短暂离车，例如拿钥匙卡、钻员工门、启动机关。
- 离车有风险，购物车可能被顾客碰走、被店员发现、被清洁机器人挡住或被保安重点检查。

## 6. 购物车系统

购物车是核心构筑载体，不只是背包。

### 6.1 基础属性

- 总格子数
- 承重上限
- 当前重量
- 当前估值
- 操控性
- 转弯半径
- 刹车距离
- 碰撞噪音
- 特殊槽位

### 6.2 容量模型

原型使用抽象格子 + 重量模型。

- 每件商品有格子占用和重量。
- 小商品占少量格子。
- 中商品占中等格子。
- 大件占大量格子。
- 超容量不能继续装入。
- 超重影响速度和操控。
- 特殊商品可占用特殊槽位，例如冷藏槽、大件固定槽。
- 原型不做俄罗斯方块式空间摆放。

### 6.3 操作手感

购物车有轻量物理感，但不能变成难控制的模拟器。

- 空车轻、快、转向灵活。
- 重车加速慢、刹车距离长、转弯半径大。
- 超重明显降低速度和操控。
- 大件可增加购物车碰撞体积。
- 撞击可撞开顾客或轻物，但制造噪音并吸引 NPC。
- 重量影响路线和撤离判断，但玩家不应频繁和输入延迟、卡碰撞或翻车物理搏斗。

### 6.4 丢弃

购物车内商品可以随意丢弃，用于纠正误拿、释放格子、减轻重量或调整撤离策略。原型阶段不设置复杂代价。后续可加入自然后果，例如丢弃物挡路、被清洁机器人推走、被店员收回或被顾客捡走。

### 6.5 购物车 UI

购物车 UI 常驻显示，不暂停游戏。

常驻摘要：

- 格子占用
- 当前重量
- 总估值
- 大件标记
- 冷藏标记
- 任务标记
- 超重、可疑、出口限制等警告

展开详情：

- 商品列表
- 单件价值
- 单件重量
- 单件格子
- 丢弃操作
- 后续可加入保险箱转移

UI 应轻量，不遮挡核心视野。

## 7. 商品系统

### 7.1 商品价值维度

商品不只提供价格，还提供策略压力。

字段建议：

- 现金价值
- 用途价值
- 格子占用
- 重量
- 风险标记
- 任务标记
- 商品类别
- 特殊槽位需求
- 是否可拆解
- 是否可研究
- 是否可交付

### 7.2 商品拿取

最新规则：商品拿取是瞬间动作，不会被打断。

- 货架 UI 中一次只拿一个商品。
- 所有商品一次输入拿取，不做二次确认。
- 满足条件时立刻入车。
- 不做拿取读条、中断、掉落或半装车状态。
- 容量不足、超重、槽位不足、工具不足等只提示原因，不自动替换已有商品。

高价值商品风险来自商品本身、位置、NPC 规则、容量重量、撤离条件或特殊需求，不来自拿取过程被打断。

### 7.3 大件商品

大件商品拿取仍瞬间完成，难点放在后果。

- 需要大量格子。
- 需要大件槽、固定架，或占用大量普通格。
- 重量高，拖慢推车。
- 可增加碰撞体积。
- 更容易被保安或店员识别为可疑。
- 可能需要货梯或装卸口撤离。

### 7.4 原型商品分类

原型先按玩法功能分类，再套真实超市外观。

1. 小而便宜：薯片、牙膏、罐头。低风险，填补剩余空间。
2. 小而贵：剃须刀、电池、保健品。价值高，可能更容易被盯上。
3. 重而普通：大米、桶装水、猫砂。价值一般，但影响推车手感。
4. 冷藏商品：冰淇淋、冻肉、冷冻披萨。需要尽快撤离或冷藏模块。
5. 大件高价：电视、微波炉、空气炸锅。高价值，高装载和撤离压力。
6. 任务商品：路由器、电池组、压缩机。推动飞碟修复或系统解锁。

## 8. 货架与扫描器

### 8.1 货架揭示

原型采用“货架类别可见，具体商品靠近后揭示”。

- 远处能看出货架类型，例如冷冻、零食、家电、日用品、仓库。
- 靠近后显示具体商品。
- 稀有商品可有弱提示，例如发光、特殊包装、被店员看守。
- 被顾客抢走或店员重新摆货后，货架状态会变化。
- 原型不需要每个商品真实摆在货架上，先用交互点和图标验证节奏。

### 8.2 外星扫描器

扫描器进入第一版原型，但不是独立复杂工具。它本质上是打开货架后的信息包装，用于解释商品 UI 和信息揭示。

打开货架后，扫描器显示：

- 商品名称
- 价值
- 重量
- 格子占用
- 风险标记
- 任务标记
- 用途是否未知

失败时，可保留已扫描到的图鉴或用途信息。后续升级可扩展为远程扫描货架、预测撤离口、识别保安路线或发现隐藏商品。

### 8.3 货架 UI 时间规则

打开货架时游戏不暂停，也不慢动作。

- NPC 继续行动。
- 倒计时继续运行。
- 警戒继续变化。
- UI 必须轻量、快速、一眼可读。
- 被 NPC 接近或警戒上升时，UI 给明显提示。

## 9. NPC 与警戒

### 9.1 NPC 定位

NPC 是规则系统的一部分，不是全部做成敌人。

- 保安：明确威胁源。看到偷拿、高噪音、禁区行为后追捕玩家。
- 店员：规则执行者。阻止未解锁商品、要求小票、呼叫保安、重新摆货。
- 清洁机器人：移动地形风险。挡路、制造湿滑地面、推动掉落商品。
- 顾客：动态障碍和机会。挡路、抢促销货、制造掩护，也可能告状。
- 经理/广播系统：阶段性压力来源。后续可用于封区、巡查、改变出口条件。
- 特殊 NPC：交易、误导、给任务，或被外星人误认为地球资源管理员。

NPC 的核心价值是改变路线、时间、噪音、视线和商品可得性。

### 9.2 冲突边界

玩家不能常规攻击或击杀 NPC，不以清场为目标。

允许：

- 推车冲撞
- 干扰
- 诱导
- 非致命道具
- 关门、绕路、制造人群混乱

NPC 受干扰后进入眩晕、调查、追逐、呼叫支援或封区等状态，不以掉血为主。

### 9.3 警戒表达

原型使用明确视锥、噪音圈和怀疑条，保证规则可读。

- 保安有可见视锥。
- 店员视锥较短，但对拿商品、员工区、大件更敏感。
- 噪音用圆形范围表达。
- 怀疑条分为注意、调查、追逐、呼叫支援。
- 顾客不追捕，但会遮挡、制造噪音或告状。
- 被发现不是立刻失败，玩家有反应窗口。

## 10. 撤离系统

撤离不是到出口立刻结束，而是短读条或条件检查。

撤离点类型：

- 普通出口：到达后短读条，约 2 到 4 秒，被追上会中断。
- 员工通道：需要钥匙卡、伪装等级或提前开门。
- 货梯/装卸口：适合大件商品，但需要等待或制造较大噪音。
- 收银口伪装撤离：需要小票或假小票，适合低警戒路线。
- 紧急出口：速度快，但会触发警报，适合最后逃命。
- 特殊出口：动态刷新，受事件影响。

撤离本身应是决策，不只是跑到终点。

## 11. 目标与任务

每轮同时存在自由搜刮和明确任务。

目标层级：

- 基础目标：尽可能带出高价值商品。
- 主线目标：为修飞碟收集模块材料。
- 支线目标：来自特殊 NPC 或外星飞船系统的误解任务。
- 本轮目标：随机小挑战，例如不触发警报、带出冷藏品、从员工通道撤离。
- 高风险目标：奖励高，但可能改变场内规则。
- 隐藏目标：通过信息类收益解锁。

目标结构要让玩家既能自由决策，也知道可追求方向。

## 12. 结算与失败

### 12.1 成功撤离

成功撤离后，购物车内商品完整结算。玩家可选择：

- 出售
- 拆解
- 交付
- 收藏研究
- 放入保险箱

普通商品可快速卖钱。稀有、大件、特殊或任务商品应保留多种用途。

### 12.2 失败

失败时主要损失本轮购物车中的商品，不损失永久强化。

保留：

- 信息类收益
- 部分任务进度
- 部分经验
- 保险箱内物品

死亡会扣除一定比例经验。失败不应纯浪费，但成功撤离仍要明显更有价值。

## 13. 局外成长

### 13.1 购物车成长

购物车承担主要构筑压力。

可成长方向：

- 车架
- 轮子
- 篮筐
- 挂钩
- 冷藏模块
- 大件固定架
- 隐蔽模块
- 一次性消耗件

### 13.2 角色成长

角色成长不能抢走购物车的核心地位。角色主要强化认知、伪装、效率和容错。

方向：

- 外星感知：识别商品价值、用途和危险标签。
- 地球伪装：降低店员或保安怀疑。
- 推车技术：改善加速、刹车、急转和窄通道操控。
- 规则误读修正：理解小票、促销、员工区、会员卡、试吃等机制。
- 应急能力：被追捕时短暂加速、减少掉货。
- 保险能力：提升保险箱格数，或降低死亡经验扣除。

避免让角色升级变成无视路线、风险和 NPC 规则的纯数值压制。

## 14. 第一版原型范围

第一版原型目标：验证单轮核心循环是否好玩。

必须包含：

- 小型固定超市布局。
- 第三人称俯视角。
- 推车状态下的人车整体控制。
- 购物车格子 + 重量系统。
- 常驻购物车 UI。
- 货架类别可见，靠近后揭示具体商品。
- 外星扫描器包装货架 UI。
- 商品瞬间拿取，一次只拿一个。
- 商品可随意丢弃。
- 6 类原型商品。
- 两类 NPC：保安和顾客。
- 视锥、噪音圈、怀疑条。
- 关店倒计时。
- 至少 2 个撤离口，其中 1 个带条件。
- 成功/失败结算。
- 一个最小升级，例如购物车容量 +1 或推车操控 +1。

暂不展开：

- 完整飞碟模块链。
- 复杂商品拆解系统。
- 多 NPC 生态全量实现。
- 完整保险系统。
- 商品形状摆放库存。
- 拿取读条和打断。
- PvP。

## 15. 待定问题

- 常驻购物车 UI 采用摘要常驻还是完整格子常驻。
- 原型中保险箱是否可在场内使用，还是只在局外使用。
- 冷藏商品在原型中是否需要品质倒计时。
- 大件碰撞体积是否进入第一版原型。
- 店员和清洁机器人是否进入第二版原型。
- 具体美术风格：写实超市、卡通超市、伪综艺节目包装，或外星人主观滤镜。
