# 超市搜打撤游戏设计记录

日期：2026-05-22

## 记录规则

每一轮设计访谈结束后，先把本轮已经达成共识的设计结论追加到本文档，再进入下一轮问题。这样即使聊天上下文过长，项目也保留一份可追溯的设计档案。

## 核心概念

这是一款以超市为主题的搜打撤游戏。玩家推着购物车在超市中抢商品，在压力下尽可能带走更高价值的货物，并在每轮结束后用带出的物品强化后续轮次。

## 背景设定

主角是一个外星人。由于飞碟故障，主角临时停靠在地球，并收看到地球上的“限时在超市里拿东西”的综艺节目。主角误以为地球上的物资就是通过这种形式获得的，于是为了修好飞碟、回到母星，开始在超市中奋斗。

## 已确认设计

### 第 1 轮：压力模型

- 初期目标是单人 PvE 撤离玩法，不先做 PvP。
- 每一轮有关店倒计时。
- 场内存在干扰玩家的 NPC，包括保安、店员、清洁机器人和其他顾客。
- 购物车同时有容量限制和重量限制。
- 商品越贵越难拿，通常也越占空间。
- 部分商品需要特殊购物车或特殊处理条件，例如冰箱、大彩电。
- 撤离口会动态变化，或需要满足某些条件才能撤离。

### 第 2 轮：策略目标

- 玩家优化的是风险调整后的收益，而不是单纯拿最贵的商品。
- 商品可以通过价值、体积、重量、拿取时间、危险度、需求条件和强化标签形成策略空间。
- 高价值商品必须有诱惑力，但也必须伴随明确成本，例如空间、重量、时间、NPC 注意力、特殊需求或更困难的撤离条件。

### 第 3 轮：失败损失

- 失败时主要损失本轮购物车中的商品，不损失永久强化。
- 失败后可以保留信息类收益，例如商品图鉴、路线情报、NPC 行为认知或撤离条件线索。
- 失败后可以保留部分任务进度。
- 失败后可以获得一部分经验，但死亡会扣除一定比例。
- 玩家拥有保险箱，保险箱内的物品不会因失败而丢失。

### 第 4 轮：成长与构筑载体

- 购物车是核心构筑载体，不只是普通容器。
- 购物车可以通过车架、轮子、篮筐、挂钩、特殊模块、隐蔽模块和一次性消耗件形成局外成长与本轮构筑。
- 玩家本人也有成长系统，但购物车应承担更主要、更具主题辨识度的构筑压力。
